# 剪切板历史 功能需求文档

> 适用项目：Electron 桌面应用（**无 Rust/原生模块编译环境**，原生能力均用 Electron API 或系统命令替代）
> 参考实现：Alfred Clipboard History
> 文档目标：交给 AI 按本文档独立完成开发，验收标准见文末

---

## 1. 功能概述

后台监听系统剪贴板，自动记录用户复制的文本、图像、文件，通过全局快捷键在屏幕中央唤起历史面板；支持搜索、分类、收藏、来源应用展示，选中条目可一键粘贴回当前应用的输入框。

## 2. 数据采集（主进程）

### 2.1 监听方式

- 主进程 `setInterval` 轮询系统剪贴板，间隔 **800ms**（Electron 无剪贴板变化事件）。
- 每次轮询读取 `clipboard.availableFormats()`，按以下**优先级**判定内容类型：

| 优先级 | 类型 | 判定与读取方式 |
|---|---|---|
| 1 | 图像 | `formats` 含 `image/*` 时 `clipboard.readImage()`，非空即图像。**必须先于文件判定**：微信/QQ 等应用复制图片时会同时写入临时文件路径，文件优先会误分类 |
| 2 | 文件 | macOS：`clipboard.readBuffer("NSFilenamesPboardType")` 解析 plist XML 中的 `<string>` 得到多文件路径；空则回退 `clipboard.read("public.file-url")`（单文件）。Windows：`clipboard.readBuffer("FileNameW")` 按 UCS-2 解码去尾部 `\0`（单文件） |
| 3 | 文本 | `clipboard.readText()`，空白串忽略 |

- **特例**：文件类型若为「单个图片文件」（扩展名 png/jpe?g/gif/bmp/webp），用 `nativeImage.createFromPath` 读取内容按**图像**入库，并记录 `sourcePath`（原文件路径）。
- 整个轮询体包 try/catch，单次失败不中断定时器。

### 2.2 去重与置顶

- 每条内容计算 hash（文本：`sha1("text:"+内容)`；图像：`sha1(PNG字节)`；文件：`sha1("files:"+路径JSON)`）。
- 模块级 `lastHash` 变量记录上次内容，未变化不处理。
- hash 已存在于库中 → 不新增，仅更新其 `lastUsedAt`（置顶）。
- 从面板复制/粘贴条目时会写回剪贴板，需先同步 `lastHash = 该条目hash` 防止轮询误判为新内容，并**显式**更新该条目 `lastUsedAt` + 广播变更事件。
- 应用启动时先读一次剪贴板初始化 `lastHash`，避免把启动前的存量内容记为新条目。

### 2.3 来源应用记录（无 Rust 方案）

- 新条目入库时获取**当前前台应用**作为复制来源，存 `sourceApp`（名称）与 `sourceAppPath`（应用路径）。
- 推荐用 npm 包 **`active-win`**（自带预编译二进制，无需编译工具链）：`(await activeWindow()).owner` 提供 `name / path`。
- 备选：macOS `osascript -e 'tell application "System Events" to get name of first process whose frontmost is true'`；Windows PowerShell + user32。
- 获取失败时静默降级（字段留空），不影响主流程。

### 2.4 存储

- SQLite 表 `clipboard_items`：

| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PK 自增 | |
| type | STRING | `text` / `image` / `files` |
| content | TEXT | 文本内容；图像为落盘 PNG 的绝对路径；文件为路径数组 JSON |
| preview | STRING | 文本前 200 字符；图像为 `宽x高`；文件为文件名列表 |
| hash | STRING（索引） | 去重用 |
| favorite | BOOLEAN 默认 false（索引） | |
| size | INTEGER | 文本字符数 / 图像字节数 / 文件个数 |
| sourcePath | STRING 可空 | 图片文件来源的原文件路径 |
| sourceApp | STRING 可空 | 来源应用名称 |
| sourceAppPath | STRING 可空 | 来源应用路径（取图标用） |
| lastUsedAt | DATETIME | 排序字段。**勿用 createdAt 排序**（ORM 对 timestamps 字段更新有特殊处理，不可靠） |
| createdAt / updatedAt | DATETIME | |

- 图像不存库：PNG 落盘到 `userData/clipboard/<hash>.png`，删除条目时若无其他引用则删文件。
- 文本超过 100,000 字符截断。
- **容量上限**：非收藏条目最多 **100** 条，超出按 `lastUsedAt` 升序删除最旧的（连带图像文件）；收藏最多 **100** 条，收藏不参与滚动清理。
- 收藏超上限时报错文案：`已达收藏历史数量上限，请阁下删除收藏历史后再进行操作`（按目标项目人设调整）。

## 3. 历史面板（独立无边框窗口）

### 3.1 窗口行为

- 全局快捷键（默认 **Alt+V**，可配置）切换显示/隐藏。
- 尺寸 **680×520**；frameless、transparent、alwaysOnTop、skipTaskbar、不可缩放。
- **首次创建时定位到鼠标所在显示器的工作区中央**；之后保持用户拖拽后的位置。
- 顶部搜索栏与分类栏空白处为拖拽区（`-webkit-app-region: drag`，输入框和按钮 no-drag）。
- 失焦自动隐藏；Esc 隐藏（**在主进程 `webContents.on("before-input-event")` 拦截**，不依赖渲染层焦点位置）。
- 每次弹出：重置搜索词与分类、聚焦搜索框、刷新列表（主进程 `webContents.send("clipboard:panel:shown")` 通知渲染层）。

### 3.2 布局

```
┌──────────────────────────────────────────────┐
│ 🔍 搜索框（即时过滤，可拖拽区）                   │
├──────────────────────────────────────────────┤
│ 全部 | 文本 | 图像 | 文件 | 收藏（分类 tab）      │
├──────────────────┬───────────────────────────┤
│ 条目列表(300px)   │ 右侧预览面板                 │
│ 序号+来源图标+摘要 │ （随选中项变化）              │
├──────────────────┴───────────────────────────┤
│ 操作提示：Enter/双击 粘贴 · 1-9 快选 · Esc 关闭   │
└──────────────────────────────────────────────┘
```

### 3.3 列表条目

- 每行：序号（前 9 条显示 1-9）+ **来源应用图标**（无来源信息时回退类型图标）+ 内容摘要 + 收藏星标。
- 文本：两行截断；若整体是色值（#hex / rgb / hsl），行尾附小色块。
- 图像：缩略图 + 名称（来源文件名；纯位图显示 `图像: 宽x高`）+ 副行 `宽x高 · 大小`。
- 文件：文件名列表（截断）。
- 排序：`favorite DESC, lastUsedAt DESC`（收藏永远置顶）。
- 来源应用图标获取：`app.getFileIcon(sourceAppPath)`；**macOS 对 .app 目录优先用 `nativeImage.createThumbnailFromPath(path, {width:64,height:64})`**（getFileIcon 对 .app 常返回通用图标）。主进程按路径缓存 DataURL。

### 3.4 右侧预览

- 文本：全文滚动展示；色值额外显示大色块 + 色码。
- 图像：大图 + 详情卡片（名称、尺寸、大小、位置=原文件路径、来源应用、复制时间精确到秒）。
- 文件：逐个显示文件名+完整路径，图片文件加缩略图（主进程按路径生成 DataURL，**勿用 file:// img src**，会被 webSecurity 拦）。
- 底部元信息栏：来源应用图标+名称 · 字符数/尺寸/文件数 · 最近使用时间。

### 3.5 键盘与鼠标交互

挂在 **document 级**监听（避免焦点在列表/空白处时失效）：

| 操作 | 行为 |
|---|---|
| ↑ / ↓ | 移动选中项（自动滚入视区） |
| Enter | 粘贴选中项 |
| 1-9 | **搜索框为空时**快选粘贴第 N 条 |
| Esc | 隐藏面板 |
| 单击 | 选中（更新预览） |
| 双击 | 粘贴 |
| 右键 | 菜单：复制 / 收藏(取消收藏) / 删除 |

### 3.6 粘贴（无 Rust 方案）

流程：写回剪贴板 → 隐藏面板（焦点回到原应用）→ 延迟 ~150ms → 模拟粘贴按键。

- macOS：`osascript -e 'tell application "System Events" to keystroke "v" using command down'`（**需要辅助功能/自动化授权**，未授权时静默失败）。
- Windows：PowerShell `[System.Windows.Forms.SendKeys]::SendWait("^v")` 或 `node-key-sender`。
- **降级策略**：模拟按键失败/不可用时仅完成复制，前端 toast「已复制到剪贴板」。
- 文件类条目粘贴时写回的是路径文本（写回文件剪贴板需额外原生能力，可作为后续项）。

## 4. 设置项

| 设置 | 默认值 | 说明 |
|---|---|---|
| 剪切板历史开关 | 开 | 关闭时停止轮询（已有数据保留），重新开启恢复 |
| 面板快捷键 | Alt+V | 录制式输入框，见下 |
| 清空历史 | — | 删除全部非收藏条目及其图像文件 |

- 快捷键录制：点击输入框后按组合键即保存为 Electron Accelerator 格式；**必须含修饰键**；主键**必须用 `event.code`（物理键位）识别**——macOS 上 `Option+字母` 的 `event.key` 是组合字符（如 Option+V → "√"），会生成无法触发的快捷键。Esc 取消录制。
- 快捷键配置变更后主进程立刻 `globalShortcut.unregisterAll()` 并重新注册；注册失败（被占用）打日志不崩溃；`will-quit` 时注销全部。

## 5. IPC 接口清单（渲染层 ↔ 主进程）

```
list(category, keyword)      查询（分类: all/text/image/files/favorite；关键词匹配 preview）
copy(id)                     写回剪贴板并置顶
paste(id) -> boolean         写回+隐藏面板+模拟粘贴，返回是否成功触发按键
setFavorite(id, favorite)    收藏/取消（超上限抛错，错误消息直接展示）
remove(id)                   删除单条（连带图像文件）
clear()                      清空非收藏
readImageDataUrl(id)         图像条目缩略图（宽 320 重采样，勿传原图）
readPathThumbnail(path)      本地图片文件缩略图（校验扩展名）
getAppIcon(path)             来源应用图标（主进程缓存）
hidePanel()                  隐藏面板
事件广播: clipboard:history:changed（增删改收藏）/ clipboard:panel:shown（弹出）
```

## 6. 边界与已知坑（务必阅读）

1. **分类判定顺序必须是 图像 > 文件 > 本文**，否则微信/QQ 复制的图片会进文件分类。
2. macOS 复制文件时剪贴板同时有 text/plain 路径文本，文件判定必须先于文本，否则文件会被记成路径文本。
3. `event.key` 在 mac Option 组合下是合成字符，快捷键录制必须用 `event.code`。
4. Esc 关闭要在主进程 before-input-event 层拦截；渲染层焦点丢失时 React 根节点收不到键盘事件。
5. 排序字段用自建 `lastUsedAt`，不要试图更新 ORM 管理的 `createdAt`。
6. 面板写回剪贴板后要同步轮询的 `lastHash`，并显式置顶，否则粘贴后条目不会移到最前。
7. 数字键快选仅在搜索框为空时生效，否则无法搜索数字内容。
8. 图像/文件缩略图一律走 IPC 返回 DataURL，渲染层直接 file:// 会被 webSecurity 拦截。
9. mac 模拟粘贴需要辅助功能授权，首次使用给用户授权引导；未授权时必须有「仅复制」降级。
10. 表结构后续加列时用 `PRAGMA table_info` 检查 + `ALTER TABLE ADD COLUMN` 手动迁移（`sync()` 不会加列）。
