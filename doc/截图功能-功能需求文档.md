# 截图功能 功能需求文档

> 适用项目：Electron 桌面应用（**无 Rust/原生模块编译环境**，原生能力用 Electron API、系统命令或脚本桥接替代）
> 参考实现：微信截图 + Snipaste（贴图）
> 文档目标：交给 AI 按本文档独立完成开发，验收标准见文末
> 配套文档：《剪切板历史-功能需求文档》（截图复制后由剪切板历史自动收录，两者联动）

---

## 1. 功能概述

全局快捷键触发截图：冻结当前屏幕画面 → 框选区域 → 对象化标注（矩形/椭圆/箭头/画笔/马赛克/文字，全部可选中再编辑）→ 输出（复制/保存/贴图钉桌面/OCR 文字识别/翻译）。

## 2. 触发与抓屏（主进程）

- 全局快捷键默认 **Ctrl/Cmd+Alt+A**（设置中可录制修改、可开关，配置变更即时重注册）。
- `desktopCapturer.getSources({types:["screen"], thumbnailSize: 显示器物理像素})` 抓取**鼠标所在显示器**的画面，按 `source.display_id === String(display.id)` 匹配目标屏，多显示器 v1 只截光标所在屏。
- **macOS 屏幕录制权限**：`systemPreferences.getMediaAccessStatus("screen")` 非 granted 时，先调一次 desktopCapturer（让应用出现在系统设置列表），再弹引导对话框，按钮跳转 `x-apple.systempreferences:com.apple.preference.security?Privacy_ScreenCapture`。授权后需重启应用才生效（系统限制）。
- 截图进行中（覆盖窗可见）忽略重复触发。

## 3. 覆盖窗与"无感打开"（关键体验）

- 覆盖窗铺满目标显示器：frameless、alwaysOnTop（**screen-saver 层级**，盖住 Dock/菜单栏）、skipTaskbar、`setVisibleOnAllWorkspaces(true, {visibleOnFullScreen: true})`。
- **禁用 mac `type:"panel"`**：NSPanel 无法挂载输入法上下文，文字工具的中文候选框永远不显示。用普通窗口。
- **无感打开三件套**：
  1. 覆盖窗**常驻复用**：应用启动约 3s 后后台预创建并加载页面（不显示）；截图结束只隐藏不销毁；每次会话主进程发 `screenshot:session` 事件驱动渲染层重置状态并拉取新画面
  2. 渲染层图片 `onLoad` 后才通知主进程显示窗口（避免先显示黑底再出画面）；加 1.5s 兜底强制显示
  3. 连续两次截图画面相同时 img 不触发 onLoad——渲染层需先清空 src 再在下一 tick 赋值，强制重载
- 渲染层以 `<img>` 全屏显示冻结画面，上面叠一个全屏 canvas（devicePixelRatio 缩放）画遮罩/选区/标注。

## 4. 框选阶段

- 十字光标，拖拽框选；全屏 45% 黑色遮罩，选区挖亮并描边框。
- 选区左上角实时显示尺寸角标 `宽 x 高`。
- 松开后进入标注阶段（选区 ≤3px 视为误触忽略）；标注阶段按 Esc 回到重新框选。
- v1 不支持框定后拖拽调整选区（重选走 Esc）。

## 5. 标注系统（对象化架构，核心）

### 5.1 对象模型

所有标注是带 id 的对象存于数组（非位图涂改）。**这是硬性架构要求**——后续每种新工具只是新增一种 kind：

```ts
BaseObj { id, color, size /*0|1|2 档位*/, rotation /*弧度，绕包围盒中心*/ }
kind: rect | ellipse | arrow  → { from: Point, to: Point }
kind: pen | mosaic            → { points: Point[] }
kind: text                    → { at: Point, value, fontSize /*实际px*/, wrapWidth }
```

### 5.2 通用交互（微信截图式）

- **单击选中**：命中检测从最上层对象开始——文字按包围盒；矩形按边框邻近（6px 容差）；椭圆按椭圆方程边线距离；箭头/画笔按线段距离；马赛克按笔宽/2+容差。旋转对象先把指针逆旋转到局部空间再判定。
- 选中态：白色虚线包围盒 + 手柄白色圆点；**工具栏自动切换到该对象对应工具**，二级面板的颜色/尺寸调整直接作用于选中对象。
- **拖拽移动只平移，绝不改变形状**（文字不重排——见 5.4）。
- **变形只通过手柄**：矩形/椭圆/画笔/文字四角缩放 + 顶部中心旋转手柄（连线+圆点，距顶边 22px）；箭头是两端点手柄直接拖动。
- **缩放穿零翻转**：缩放手势开始时**固化锚点（对角）与原始形状快照**，此后每帧从快照计算——指针拖过锚点时图形自然镜像翻转（画笔用带符号缩放系数），且无逐帧累积误差。
- **光标反馈**：悬停对象→move；角点→按含旋转的实际方向选 nwse/nesw/ns/ew-resize（方向角 45° 一档）；旋转手柄→内嵌 SVG 圆形箭头光标。
- **画完不自动选中**（保持连续绘制效率），需单击选中后再调整。
- Del/Backspace 删除选中；Cmd/Ctrl+Z 撤销（移除最后一个对象）；Esc 分级：文字编辑中→退出编辑 → 有结果面板→关面板 → 有选中→取消选中 → 标注中→重新框选 → 框选中→退出截图。
- **StrictMode 警告**：对象创建等副作用严禁写进 setState 更新函数（React 双调用会产生重复对象），读 ref 后顺序执行。

### 5.3 各工具规格

| 工具 | 绘制 | 备注 |
|---|---|---|
| 矩形 | 拖出外接框描边 | 线宽三档 2/3/5 |
| 椭圆 | 外接框内切椭圆 | 同上 |
| 箭头 | 直线+箭头头部（头长随线宽） | 不支持旋转（端点可达同效果） |
| 画笔 | 自由折线 | 同线宽档位 |
| 马赛克 | 宽笔刷涂抹（14/22/34 三档），显示像素化效果 | 见 5.5 |
| 文字 | 点击放置输入框 | 见 5.4 |

### 5.4 文字工具（重点打磨项）

- 点击选区内弹出**多行输入框**：样式与选中态一致（白虚线框+四角白点）；宽度随内容增长，达到**选区右缘自动换行**（中英文按字符折行，canvas 渲染与编辑框用同一折行函数保证所见即所得）；内容贴近选区底部时**输入框自动上移**保证可见，上移后位置写回对象。
- **换行宽度 wrapWidth 在提交时固化**：移动对象不重排；只有再次进入编辑才按当前位置重新做边缘检查。
- 再编辑：**双击文字**，或**已选中状态下再单击**；原内容/颜色/字号带回；Esc 放弃修改；点击外部（blur)提交；清空内容=删除对象。
- 字号：工具栏三档预设（14/18/26px）+ **角点连续缩放**——横向拖调 wrapWidth（重排折行）、纵向拖调字号、对角同时调（不做镜像，字号下限 8px）。注意不能用对角线距离做等比缩放——文字包围盒宽扁，纵向拖拽时系数几乎不变，表现为"高度调不动"。
- 支持旋转；编辑旋转过的文字时输入框用 CSS transform 同角度显示。
- **mac 输入法三件套**（缺一不可）：① 窗口不能是 panel 类型；② 进入输入时激活应用（`app.focus({steal:true})`）；③ 输入时把窗口层级从 screen-saver 临时降到 floating（screen-saver 比输入法候选窗层级高，会盖住候选框），退出输入恢复。
- 放置输入框的 mousedown 必须 `preventDefault()`（否则手势收尾焦点转移导致输入框刚挂载即 blur 销毁），并在挂载后 ~30ms 显式 focus。

### 5.5 马赛克

- 画面加载时生成**全屏像素化底图**：缩小 12 倍再放大（`imageSmoothingEnabled=false`）。
- 马赛克对象=宽笔刷路径；渲染时以像素化底图为 canvas pattern 沿路径描边——像素块与底图天然对齐；导出合成时在缩放变换下 pattern 自动对齐（pattern 受当前 transform 影响）。
- 是叠加涂层非破坏性修改：可选中、拖移（露出原图）、缩放、删除。

### 5.6 工具栏与二级面板

- 主工具条位于选区下方（放不下翻到上方）：6 个工具 + 分隔 + 撤销/取消/保存/贴图/OCR/翻译/完成(✓)。
- **每个工具有自己的二级面板**（微信式）：带小箭头、精确锚定在当前工具按钮正下方（工具条翻上方时面板翻到上方、箭头朝下）。内容：文字=字号三档(A A A)+颜色；马赛克=笔刷粗细三档；其余=线宽三档(圆点)+颜色。颜色三选（红 #FF3B30 / 黄 #FFCC00 / 蓝 #0A84FF）。
- 工具条/面板上的 mousedown 要 stopPropagation，避免触发画布手势。

## 6. 输出动作

- **完成/复制**（✓ / Enter / 双击空白）：合成 PNG 写入剪贴板（剪切板历史自动收录），关闭截图。合成=按物理像素裁剪底图+以同一 drawObj 函数叠加标注（坐标系换算：ratio = 底图像素宽/窗口 CSS 宽）。
- **保存**：系统保存对话框，默认文件名 `<产品名>截图-YYYYMMDD-HHmmss.png`、默认图片目录；保存后同时复制。
- **贴图**：见第 7 节。
- **OCR**：识别选区**原始画面（不含标注，标注会污染识别）**。macOS 无 Rust 方案：`osascript -l JavaScript` 用 JXA 桥接 Vision 框架（`ObjC.import('Vision')`，VNRecognizeTextRequest，精确模式+语言校正，语言优先级 zh-Hans/zh-Hant/ja/en），临时 PNG 入参、stdout 回传文本；备选 Tesseract.js（跨平台但中文准确率一般）。Windows 预留接口（reject 特定错误码，前端显示"暂不支持"）。结果面板显示在选区上方：加载态/结果文本（可选中局部复制）/一键复制全部/空结果与错误提示，Esc 优先关面板。
- **翻译**：OCR → LLM 翻译（用目标项目现有的模型 API；提示词要求只输出译文、保持分行）。方向自动判定：CJK 字符占比>50% 视为中文为主→译英文，否则→译中文。共用 OCR 结果面板（标题区分）。

## 7. 贴图（Snipaste 式）

- 截图（含标注）**原位**钉在桌面：贴图窗口位置=显示器原点+选区坐标，大小=选区，视觉上画面"留在"屏幕上。
- 独立无边框窗口，可同时多张：alwaysOnTop(floating)、skipTaskbar、有阴影、细边框标识。
- 交互：左键拖拽移动（渲染层 mousedown 记录偏移，mousemove 用 e.screenX/Y 经 IPC setBounds）；滚轮缩放 0.2–3 倍（以中心缩放）；**右键弹主进程原生 Menu**：复制/保存/关闭贴图/关闭全部贴图；Esc/双击关闭。
- 若有"老板键"类全局隐藏功能，贴图需联动隐藏/恢复（恢复用 showInactive 不抢焦点）。

|---|---|---|

## 9. IPC 接口清单

```
screenshot.getCapture()            冻结画面 DataURL
screenshot.ready()                 渲染层画面加载完成 → 主进程显示覆盖窗
screenshot.setInputMode(active)    文字输入模式（激活应用/调层级）
screenshot.finish(dataUrl)         复制并结束
screenshot.save(dataUrl)           保存对话框
screenshot.pin(dataUrl, selection) 贴图
screenshot.ocr(dataUrl)            文字识别
screenshot.translate(dataUrl)      OCR+翻译 → {source, translation}
screenshot.cancel()                取消
stickers.getImage/move/setScale/showMenu/close
事件: screenshot:session（主进程 → 渲染层，开始新会话）
```

## 10. 边界与已知坑（务必阅读，均为实测踩过）

1. 覆盖窗禁用 mac panel 类型（输入法）；剪切板那种小面板可以用 panel，但有文字输入的窗口不行。
2. screen-saver 层级会盖住输入法候选窗，输入时必须临时降层级。
3. 无感打开依赖窗口常驻复用 + onLoad 后显示 + src 清空重赋（三者缺一会有黑屏/缩放感/画面不刷新）。
4. 缩放锚点必须手势开始时固化，否则穿零时锚点跳变、无法翻转，且有累积误差。
5. 文字缩放不能用对角线等比（宽扁包围盒纵向不灵敏），用横纵独立系数。
6. React StrictMode 双调用：副作用出 setState 更新函数。
7. 文字放置 mousedown 要 preventDefault + 延迟显式 focus，否则输入框秒消失。
8. OCR 用无标注底图；合成用同一 drawObj 保证导出与所见一致。
9. 马赛克 pattern 与底图对齐依赖"pattern 受当前变换影响"这一特性，合成时勿用 setTransform 之外的方式偏移。
10. mac 屏幕录制授权后必须重启应用；dev 模式下授权对象可能是 Electron 或终端。
11. 命中检测/手柄检测都要先做旋转逆变换；手柄可能在选区外，检测顺序：手柄 > 对象 > 空白新建。
12. 主进程若单独维护 i18n 词表，权限对话框/贴图菜单等主进程文案的 key 必须加到主进程词表。
